## PrincГӯpios Chave

1. **Route Handlers como adaptadores** вҖ” Apenas parseiam request, chamam Use Case, retornam response. Sem lГіgica.
2. **Use Cases como orquestradores** вҖ” Toda regra clГӯnica vive aqui. TestГЎveis de forma isolada.
3. **Repository Pattern** вҖ” ImplementaГ§Гөes Prisma ficam em `/infra`. Use Cases dependem de interfaces.
4. **Multi-tenant** вҖ” Todo modelo clГӯnico tem `userId`. Toda query filtra por `userId`.
5. **ValidaГ§ГЈo nas bordas** вҖ” Zod valida input no Route Handler. Nunca confiar em dados sem validar.
